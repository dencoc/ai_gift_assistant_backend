# Требования к разработке проекта AI Gift Assistant Backend

## 1.1 Цели и задачи

### Цель работы

Получение навыков разработки программного кода в соответствии с требованиями к качеству, структуре и
оформлению. Реализация полнофункционального RESTful API для системы помощника по подбору подарков с
интеграцией искусственного интеллекта.

### Задачи, которые необходимо выполнить

1. Выбрать предметную область проекта
2. Изучить требования и рекомендации по оформлению кода
3. Описать ход работы и архитектуру решения
4. Сформулировать краткий вывод о выполнении работы

## 1.2 Критерии приемки работ

Работа считается успешно выполненной при соблюдении следующих критериев:

1. **Реализован проект**
    - Все функциональные требования реализованы
    - API полностью работоспособна
    - Интеграции с внешними сервисами функционируют корректно

2. **Соблюдены требования объектно-ориентированного программирования**
    - Правильное разделение ответственности (SRP)
    - Использование наследования и полиморфизма где необходимо
    - Инкапсуляция данных и методов
    - Низкая связанность компонентов

3. **Соблюдены требования по оформлению кода**
    - Единообразный стиль кодирования (ESLint конфигурация)
    - Правильная типизация (TypeScript)
    - Наличие комментариев и документации
    - Следование принципам SOLID

## 1.3 Выбор предметной области

### Выбранная предметная область

**AI Gift Assistant Backend** — реализация серверной части приложения для интеллектуального подбора
подарков на основе искусственного интеллекта.

**Обоснование выбора:**

- Актуальность применения AI в бытовых приложениях
- Практическое применение в реальной жизни
- Возможность интеграции с различными сервисами

## 1.4 Описание проекта

### 1.4.1 Цели и задачи, которые решает проект

**Основные цели:**

- Обеспечить пользователям инструмент для интеллектуального подбора подарков
- Интегрировать возможности искусственного интеллекта для улучшения рекомендаций
- Предоставить удобный интерфейс взаимодействия через Telegram бота
- Хранить и управлять информацией о событиях и пользователях

**Решаемые задачи:**

- Аутентификация и управление пользователями
- Управление событиями выбора подарков
- Сохранение истории сообщений и чатов
- Обработка файлов (фотографии)
- Отправка уведомлений о предстоящих событиях
- Интеграция с AI сервисами для рекомендаций
- Интеграция с Telegram-ботом

### 1.4.2 Сфера применения

- **Целевая аудитория:** Пользователи, нуждающиеся в помощи при выборе подарков к событиям
- **Платформы:** Web API (REST), Telegram Bot
- **Основное использование:** Помощь при выборе подарков к дням рождения, юбилеям, праздникам
- **Дополнительные возможности:** Сохранение истории выборов, рекомендации на основе предпочтений

### 1.4.3 Язык программирования и технологии

#### Backend технологии

- **Язык программирования:** TypeScript
- **Runtime среда:** Node.js
- **Web фреймворк:** Express.js
- **Базы данных:**
    - PostgreSQL (основная база данных)
    - Redis (кеширование, сессии)
- **Тестирование:** Jest
- **Интеграции:**
    - AI Chat API (для рекомендаций)
    - Telegram Bot API

#### Инструменты разработки

- **Контейнеризация:** Docker, Docker Compose
- **Линтинг:** ESLint
- **Управление типами:** TypeScript
- **Package Manager:** npm

#### Архитектурные паттерны

- **Model-View-Controller (MVC):** Инкапсуляция логики в контроллерах, моделях и сервисах
- **Service Layer:** Отделение бизнес-логики от контроллеров
- **Middleware паттерн:** Обработка аутентификации, валидации, ошибок
- **Repository паттерн:** Абстракция работы с базой данных

### 1.4.4 Структура проекта

```
src/
├── app.ts                 # Главное приложение Express
├── api/                   # Внешние API интеграции
├── controllers/           # Обработчики запросов
├── models/               # Модели данных и ORM операции
├── services/             # Бизнес-логика
├── routes/               # Маршруты приложения
├── middlewares/          # Middleware функции
├── schemas/              # Валидационные схемы
├── types/                # TypeScript типы
├── lib/                  # Утилиты и библиотеки
├── jobs/                 # Фоновые задачи (scheduler)
├── telegram/             # Интеграция с Telegram
├── utils/                # Вспомогательные функции
└── tests/                # Тесты
```

### 1.4.5 Ключевые модули и функции

#### Основные модули

- **User Management:** Регистрация, аутентификация, управление профилем
- **Chat System:** Сообщения с AI ассистентом, история чатов
- **Event Management:** Управление событиями (дни рождения, праздники)
- **Photo Management:** Загрузка и хранение фотографий
- **Message Service:** Сохранение и управление сообщениями
- **Telegram Integration:** Взаимодействие через Telegram бота
- **AI Integration:** Использование AI для рекомендаций подарков

#### Фоновые задачи

- Напоминания о предстоящих событиях
- Регулярная очистка данных
- Обновление кеша

## 1.5 Требования к оформлению кода

### 1.5.1 Стиль кодирования

- **Язык кода:** TypeScript со строгой типизацией (`strict: true`)
- **Форматирование:** Автоматическое форматирование через Prettier
- **Линтинг:** Соблюдение правил ESLint конфигурации проекта
- **Соглашения об именовании:**
    - Переменные и функции: camelCase
    - Классы и интерфейсы: PascalCase
    - Константы: UPPER_SNAKE_CASE
    - Приватные свойства: префикс `_`

### 1.5.2 Документирование кода

- Наличие JSDoc комментариев для всех публичных функций и методов
- Описание параметров и возвращаемых значений
- Примеры использования для сложных функций
- Описание исключений, которые могут быть выброшены

### 1.5.3 Требования SOLID принципов

- **Single Responsibility:** Каждый класс/функция отвечает за одно
- **Open/Closed:** Открыто для расширения, закрыто для модификации
- **Liskov Substitution:** Подтипы должны корректно заменять базовые типы
- **Interface Segregation:** Специфичные интерфейсы вместо универсальных
- **Dependency Inversion:** Зависимость от абстракций, а не от реальных объектов

## 1.6 Требования к функциональности

### 1.6.1 Функциональные возможности

#### Модуль пользователей

- Регистрация новых пользователей
- Аутентификация (логин/выход)
- Обновление профиля пользователя
- Управление паролем и восстановление аккаунта
- Загрузка аватара пользователя

#### Модуль чатов

- Создание новых чатов с AI
- Получение истории сообщений в чате
- Удаление чатов
- Получение списка всех чатов пользователя

#### Модуль сообщений

- Отправка сообщений в чат
- Получение сообщений с ответами от AI
- Удаление сообщений
- Отметка сообщений как прочитанных

#### Модуль событий

- Создание событий (дни рождения, юбилеи, праздники)
- Редактирование информации о событиях
- Удаление событий
- Получение списка предстоящих событий
- Архивирование прошедших событий

#### Модуль фотографий

- Загрузка фотографий в профиль
- Получение списка фотографий
- Удаление фотографий
- Ограничение размера и формата файлов

#### Модуль Telegram

- Привязка аккаунта к Telegram боту
- Отправка сообщений через бота
- Получение уведомлений в Telegram
- Управление настройками бота

### 1.6.2 Требования к API

- **Формат:** JSON
- **Версионирование:** API v1 в URL-пути
- **Коды ответов:**
    - 200 OK: успешный запрос
    - 201 Created: ресурс создан
    - 400 Bad Request: некорректный запрос
    - 401 Unauthorized: требуется аутентификация
    - 403 Forbidden: доступ запрещен
    - 404 Not Found: ресурс не найден
    - 500 Internal Server Error: ошибка сервера

- **Обработка ошибок:** Стандартный формат ошибок с кодом и описанием

## 1.7 Требования к безопасности

### 1.7.1 Аутентификация и авторизация

- JWT токены с временем жизни
- Refresh токены для продления сессии
- Хеширование паролей (bcrypt или аналог)
- Защита от CSRF атак

### 1.7.2 Защита данных

- HTTPS/TLS для передачи данных
- Валидация всех входных данных
- Санитизация данных перед использованием в БД
- Защита от SQL injection
- Ограничение по времени для токенов

### 1.7.3 Логирование и мониторинг

- Логирование всех важных операций
- Отслеживание ошибок и исключений
- Мониторинг производительности
- Логирование попыток несанкционированного доступа

## 1.8 Требования к тестированию

### 1.8.1 Покрытие тестами

- Минимум 70% покрытие юнит-тестами
- Тестирование основной бизнес-логики
- Интеграционные тесты для API endpoints
- E2E тесты критических пользовательских сценариев

### 1.8.2 Фреймворк тестирования

- **Unit Testing:** Jest
- **Assertions:** Jest expect API
- **Mocking:** Jest mocks и @testing-library/jest-dom
- **Coverage:** Istanbul/nyc

### 1.8.3 Типы тестов

- **Unit тесты:** Для отдельных функций и методов сервисов
- **Integration тесты:** Для контроллеров и endpoint'ов
- **Mock tests:** Для внешних зависимостей (AI API, Telegram)

## 1.9 Требования к развертыванию

### 1.9.1 Containerization

- Проект должен запускаться в Docker контейнерах
- Наличие Dockerfile для backend
- Docker Compose для локальной разработки и тестирования
- Многоступенчатая сборка для оптимизации размера образа

### 1.9.2 Переменные окружения

- Использование `.env` файлов для конфигурации
- Поддержка разных окружений (development, testing, production)
- Безопасное хранение чувствительных данных

### 1.9.3 Миграция БД

- Все изменения схемы БД должны быть версионированы
- Скрипты миграций для PostgreSQL
- Возможность отката миграций
